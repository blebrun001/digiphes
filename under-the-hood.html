IPHES-CERCA Collections Digitization

1) RDF architecture for laboratory-wide digitization

IPHES-CERCA collections digitization requires an architecture where all laboratory data (inventory, analyses, provenance, conservation, and digital assets) are represented in RDF.

RDF (Resource Description Framework) models information as subject-predicate-object triples. Serialized in Turtle and queried with SPARQL, this graph model integrates heterogeneous datasets with explicit semantic traceability.

2) CIDOC-CRM as a shared semantic backbone

Because the laboratory manages highly diverse collections, a common reference model is required to ensure semantic consistency and cross-domain interoperability.

CIDOC-CRM provides that backbone by defining stable classes and properties (for example E22, E53, P46, P108) to align objects, contexts, events, and actors in a single RDF graph.

This alignment enables unified querying and reasoning across collections while preserving domain-level specificity.

3) CRM extensions, ontologies, and FAIRness

To capture collection-specific requirements, the core model is extended with CIDOC-CRM extensions such as CRMarchaeo, CRMsci, and CRMgeo.

The stack also uses controlled vocabularies and reference ontologies (for example Getty AAT, PeriodO, UBERON, Darwin Core), published in RDF/SKOS/OWL.

Persistent URIs, provenance-aware named graphs, SPARQL querying, and SHACL validation collectively reinforce FAIR principles (Findable, Accessible, Interoperable, Reusable).
